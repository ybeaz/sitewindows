About

This is an SPA service for creating a multi-task “windows” environment on one tab of the browser. It revolutionizes the concept of a personal cabinet and allows users to organize information at their own discretion, rather than follow the hierarchy of sites or the scenarios of UX developers. Users can open and reorder in modal windows shopping carts, price lists, comparison lists, reports, tasks, offers, messages, notes, etc. This is an open source project that is built around reusable components, with a high level of visualization, encapsulation of functions, polymorphism and suggests flexible user stories or a combination thereof.:
  1. A user selects and can run different applications in the same time in the browser tab
  2. A system works as online service, can be added to any web page with javascript code. The system can add applications for users over time. It happens on the server side
  3. The applications can consist of parts (windows) and builds a tree hierarchy such as an nesting doll. Examples from daily life: Mission-Target-Project-Task-Step Corporation-Unit-Division-Department-Team-Employee Solution-Product-Feature
  4. Every application and its level of hierarchy provides the user with a catalog of objects. The user is able to manipulate these object by his/her discretion: to create, move, delete, archieve, minimize, restore, assign status, filter, color, save, open by name, etc. Examples from daily life: Stickers, Tasks, Notes, Categories, Project parts, Lines of development, etc.
  5. Every application sets its own environment, object sets and features that reflect its nature. Examples of applications: problem solving tools (TRIZ), strategic planning (ToP, SSP), task management (GTD), employees' management and organization, etc.
  6. Users can work together by exchanging their works, providing access to their works, combining works, working online in real time mode

Features

UI/UX features:
  • Authentication, authorization
  • Applicatioins: add, remove
  • Projects: create, open, save, delete
  • Cards:
    • CRUD: create, read, update, delete
    • Layout: move, remember positions
    • Types: 5 types in Top Target app
    • Workflow: create, group, duplicate in "Now", archieve in "Done"
    • View: resize, max, min, restore
    • Typography: markdown formatting
  • Parse and copy content into Clipboard
  • Local storage: save, clean, restore
   
Technology architechture:
  • View layer:
    • ReactJS Smart and dumb components
    • ReactJS provider of context
    • MapStateToProps() and mapDispatchToProps()
    • ReactJS provider of context
    • ReactJS HOC (High Order Components)
  • Data layer:
    • Redux store, cofigureStore
    • Reducers, combineReducers
    • Selectors
    • Middleware
  • Side effects:
    • Redux-saga: fork, all, call, put
  • Comminication layer: fetch, promice, async
  • Shared:
    • ES6: generators, proxy, etc.
    • Recursive functions
    • LESS css processor
    • Css flex-containers
  • Unit testing:
    • Jest, enzyme
  • Server side:
    • Php 7.2
    • API on php
  • Databases:
    • MySQL for athentication and authorization
    • MongoDB for data storage
